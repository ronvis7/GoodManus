GoodManus AI Agent System Architecture

Planner-ReAct Multi-Agent Framework with Clean Architecture, Docker Compose Orchestration

Multi-Agent Flow

  • · PlannerAgent: Task decomposition → Plan
  • · ReActAgent: Step-by-step execution
  • · PlannerReActFlow: Orchestrate IDLE → PLANNING → EXECUTING → UPDATING → SUMMARIZING
  • · Memory compaction per step to prevent context corruption
  • · Max 100 iterations, 3 retries per LLM/tool call

Protocol & Integration

  • · MCP: stdio / SSE / streamable HTTP transports
  • · A2A: Agent Card discovery + JSON-RPC 2.0 invoke
  • · Tool naming: mcp_{server}_{tool} prefix isolation
  • · AsyncExitStack lifecycle management
  • · Redis Stream: input/output message queue

Deployment & Scaling

  • · Docker Compose: 6 services + bridge network
  • · PostgreSQL 16 + Redis 7.2 health-checked
  • · Nginx reverse proxy with WebSocket support
  • · Docker-in-Docker sandbox with TTL
  • · COS cloud object storage for file persistence
